Fifth iteration of Effective Messaging Toolkit for Primary Care.

Iterations made following an NHS.UK content crit and subsequent feedback:

Message senders toolkit checklist

Navigation and lede paragraph
  • Changed page title and left hand nav on content guide index page to ‘How to send effective messages to patients’.
  • Changed lede paragraph to:

    • This guide is for people who create and send messages to patients in NHS primary and secondary care.

    Specifically, if you're a GP practice manager, receptionist or work in administration support and you need to create and send messages to patients as part of your role, this guidance will help you to make them more effective.

1. Choose the right communication channel
  • Changed link text to 'Find out more about choosing the right communication channels for your messages.
2. Make sure messages are accessible and inclusive
  • In 'Essential', updated bullets 1 and 2 to add a link and examples of accessible formats.
  • In 'Best practice', updated bullet 1 to add a link to NHS content guide.
4. Make sure messages are appropriate and relevant
  • In 'Essential', added new bullet and a link for consent if a message is considered direct marketing and added an example of a 'specific group of people'.
  • In 'Best practice', added link to voice and tone guidance and add content to clarify adding a banner or a social post is instead of a message.

Communication channels for messages

General changes
  • Updated any reference to 'app messages' to 'NHS App messages' so more specific and clear which app we're referring to.
  • Added some links to NHS content and voice and tone guide when they are referenced.
  • Removed some anchor links to other sections on the same page to reduce confusion.
Introduction
  • Re-worked intro to make this section clearer by adding an example of fallback logic or removing the 2nd paragraph - also made it clear this section includes information on how to write messages for each channel.
  • Moved 'It is important to understand the costs and benefits of each channel.' to a new paragraph.
  • Removed 'Avoid sending the same message using more than one channel at the same time. This can overwhelm patients. It also increases costs for the NHS.'
Send messages that patients can understand
  • Changed heading title to 'Send messages in a format that patients can access'.
App messages
  • Changed heading title to 'NHS App messages'
  • Moved ‘Your messaging service supplier may already offer NHS App messaging.’ and 'Find out more about getting started with NHS App messaging (NHS Digital)' to intro section.
How to send messages through the NHS App
  • Removed bulleted links and added link text back in as full sentences.
If a patient does not see an App message
  • Changed title to ‘If a patient does not open an NHS App message’.
  • Changed ‘see’ to ‘open’ in main text.
  • Added more context to cover different supplier settings.
Keep NHS App messages as short as possible
  • Changed paragraph to be consistent with section above and the wording in the text messages section on making text messages shorter.
Text messages (SMS)
  • Removed anchor links as they’re confusing and there are already links to these sections in the nav.
  • Removed the word ‘more’ and added ‘if they’re able to’ about the NHS App.
  • Added extra context around keeping contact details up to date.
  • Removed content about running reports.
Links in text messages
  • Removed whole section.
